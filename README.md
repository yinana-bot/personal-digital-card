# personal-digital-card
个人数字名片页

## 作品简介
这是一个个人数字名片网页，采用现代化的卡片式设计，展示个人信息、联系方式、自我介绍、兴趣爱好和技能标签等内容。页面具有响应式布局，适配电脑和手机设备。

## 页面主要内容
- **头部区域**：个人头像、姓名、学号、专业、个性签名
- **联系方式**：邮箱、电话、GitHub外部链接
- **自我介绍**：100字以上的个人介绍
- **兴趣爱好标签**：3个及以上的兴趣标签
- **技能标签**：3个及以上的技能标签
- **底部版权信息**

## 文件结构
- index.html - 主页面文件
- style.css - 样式文件
- images/ - 图片文件夹
- avatar.png - 个人头像
- README.md - 说明文档

GitHub Pages 访问链接：https://yinana-bot.github.io/personal-digital-card/

## 最能体现“软件作品”的一点
1. 模块化文件组织：HTML、CSS、图片资源分离存放，结构清晰
2. 响应式设计：通过CSS媒体查询适配不同屏幕尺寸
3. 可维护性：样式按功能模块划分，便于后续修改和扩展
4. 交互效果：标签悬停动画、链接状态变化等细节

## AI辅助开发说明

### 是否使用AI
是

### 使用的工具
- DeepSeek（对话式AI辅助）
- ChatGPT（代码结构参考）

### 关键提示词
1. 如何实现响应式布局，让卡片在手机上也能正常显示？
2. 请提供CSS样式，让页面看起来现代简洁，使用渐变背景和圆角设计
3. 帮我润色一下自我介绍

### AI主要帮助的部分
1. HTML结构框架设计
2. CSS样式代码生成（渐变背景、圆角卡片、响应式布局）
3. 文件结构规划建议
4. 自我介绍文本润色

### 对AI结果做的修改
1. 调整了颜色方案，使用更协调的紫色渐变
2. 修改了标签样式，增加悬停动画效果
3. 优化了间距和布局
4. 删除了部分冗余CSS代码
5. 将自我介绍改为中性客观的表述

### AI最不可靠的一点
AI生成的颜色方案有时不符合个人审美，需要手动调整；自我介绍文本偏向夸张叙述的风格，与实际情况不符，需要重新修改；代码兼容性方面可能遗漏某些浏览器的适配写法。
